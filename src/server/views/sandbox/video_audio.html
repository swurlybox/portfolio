For large media files, typically you would store them in a dedicated storage service like an Amazon S3 Bucket or Google Cloud, and then access them through their dedicated API.

You wouldn't really store them in the project directory because:

  1. 1. You'd be exposing large files in the repository, making the project incredibly bulky and time consuming to port to another environment.
  2. 2. If this were deployed on an EC2 instance with limited resources, the media files would hog all the storage, which could possibly lead to a server crash.
  3. 3. And probably many other reasons that I'm currently too tired or stupid to think of...

However, for a small project such as this, storing the video locally on the computer is fine

Note that if you were to try and clone this public repo, you wouldn't have access to the video file

as its stored locally on MY computer and not the repository

Whats interesting here is the controls attribute in the html video tag, which provides a default set of video playback controls
